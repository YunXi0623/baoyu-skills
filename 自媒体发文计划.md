# 自媒体发文计划

## 一、整体策略

### 定位

- 身份：测试工程师 + AI 产品副业
- 目标：公域引流 → 微信私域 → 卖 AI 产品（账号、Token、算力）
- 平台：小红书、微信公众号、X (Twitter)

### 核心理念

- **一鱼多吃**：一个选题，一次创作，三次改编
- **输入驱动输出**：不靠日历表规划，靠每天真实使用 AI 的过程积累素材
- **你的日常就是别人的教程**

---

## 二、标准操作流程（SOP）

每次创作内容，按以下流程执行：

```
选题 → 公众号写作（一站式：撰写+调用 baoyu-humanize 润色+审核+配图+发布） → 小红书适配 → X 适配 → 发布
```

### 完整技能链路

| 步骤 | 技能 | 说明 | 适用平台 |
|------|------|------|---------|
| 1. 找选题 | baoyu-trend-scout | 搜索热点，推荐 3 个选题方向 | 全平台 |
| 2. 写长文 + 公众号发布 | baoyu-wechat-mp-writer | 一站式：选题→撰写→调用 baoyu-humanize 去AI味→违禁词审核→配图→封面→转HTML→发布草稿箱 | 公众号 |
| 3. 小红书适配 | baoyu-xhs-images（内置审核+润色） | 把母版拆成 3-6 张图片，文案自动审核润色 | 小红书 |
| 4. X 推文适配 | baoyu-x-copywriter（内置审核+润色） | 从母版提炼短推文 | X/Twitter |
| 5. 发布推文 | baoyu-post-to-x | 发布推文到 X | X/Twitter |

### 关键规则

- **审核和润色策略**：baoyu-wechat-mp-writer 会在流程内调用 `baoyu-humanize` + `baoyu-content-review`；baoyu-xhs-images、baoyu-x-copywriter 仍维持各自内置的审核与润色链路
- **公众号文章即母版**：baoyu-wechat-mp-writer 产出的长文作为母版，供小红书和 X 改编
- **先长后短**：先写公众号长文，再往短了改编，效率最高

## 三、一鱼多吃执行流程

### 创作流程（每次约 1 小时）

```
Step 1（20 分钟）：写公众号长文（母版，最完整的内容）
Step 2（15 分钟）：从长文里提炼要点 → 做成小红书图文
Step 3（5 分钟）： 从长文里摘金句 → 发 X
Step 4（5 分钟）： 三个平台分别发布
```

先写最长的，再往短了删，比分别写三篇快得多。

### 改编示例

选题："用 Claude 写自动化测试用例"

| 平台 | 改编方式 | 内容形态 |
|------|---------|---------|
| 小红书 | 截图为主，步骤拆成卡片图，标题夸张点 | "震惊！5分钟写完50条测试用例" + 6张步骤图 |
| 公众号 | 完整教程，带 prompt、带代码、带思考过程 | 2000字深度文 |
| X | 提炼核心亮点，1-2句话 + 截图 | "Today I used Claude to generate 50 test cases in 5 min. Here's the prompt ↓" |

## 四、各平台内容策略

### 小红书

**内容分类（5 种轮着发）**：

| 类型 | 标题模板 | 示例 |
|------|---------|------|
| 效率震撼型 | "用 AI xxx，从 X 小时缩短到 X 分钟" | "用 Claude 写测试用例，2小时的活5分钟搞定" |
| 教程保姆型 | "手把手教你 xxx（附步骤）" | "ChatGPT Plus 合租指南，月均不到20块" |
| 对比测评型 | "xxx vs xxx，到底该选哪个？" | "Claude vs ChatGPT，写代码到底谁更强？" |
| 热点追踪型 | "xxx 刚更新了！这个功能太强了" | "GPT-4o 免费开放！3个隐藏玩法你得知道" |
| 场景种草型 | "我用 AI 做了 xxx，效果炸裂" | "用 Midjourney 给公司做了套海报，老板以为请了设计师" |

**要点**：
- 图文为主，3-6 张图
- 标题情绪化、夸张
- 300 字以内
- 发布时间：晚上 8-10 点

**引流话术**（评论区引导，不要正文放微信号）：
- "需要的宝子扣 1，私信发你"
- "整理了完整教程，评论区自取"

### 微信公众号

**内容分类**：

| 类型 | 频率 | 作用 |
|------|------|------|
| AI 工具深度测评 | 每周 1 篇 | 建立专业形象 |
| 实操教程 | 每周 1 篇 | SEO 长尾流量 |
| 行业资讯解读 | 热点时发 | 时效性引流 |
| 个人故事/副业分享 | 每月 1-2 篇 | 人设建设，拉近距离 |

**选题公式**：
```
痛点 + 解决方案 + 结果

例：
痛点：写周报太痛苦
方案：用 ChatGPT 的 prompt 模板
结果：10 分钟搞定，还被领导夸了
```

**标题模板**：
- "作为程序员，我靠 AI 副业月入 XXXX 的全过程"
- "2026 年最值得买的 X 个 AI 工具（亲测）"
- "ChatGPT Plus 到底值不值？用了半年说句实话"
- "测试工程师转型 AI，我走过的弯路和捷径"

**要点**：
- 1500-3000 字深度文
- 开头 3 行必须抓住注意力
- 多用小标题、粗体、分隔线
- 文末固定放引导（加微信/进群）
- 发布时间：早上 8 点或午休

### X (Twitter)

**内容分类**：

| 类型 | 格式 | 示例 |
|------|------|------|
| AI 新闻速递 | 1-2 句话 + 截图 | "Claude 刚更新了 xxx 功能，实测效果 ↓" |
| Prompt 分享 | 直接贴 prompt + 效果 | "这个 prompt 让 GPT 写出的代码直接能跑 ↓" |
| 个人观点 | 短评 + 引发讨论 | "用了3个月 Claude，说说跟 ChatGPT 最大的区别..." |
| Thread 长帖 | 系列推文 | "我测试了 10 个 AI 编程工具，排名如下 🧵" |
| Meme / 轻松内容 | 截图 + 吐槽 | AI 翻车截图、有趣对话 |

**要点**：
- 极短，280 字符
- 中文或英文均可，英文受众更大
- 每周 1 条 Thread（深度内容，涨粉利器）
- 多互动、多 reply 大 V 的帖子
- 蹭热点要快（AI 新闻出来 1 小时内发）

## 七、注意事项

### 内容层面

1. **不要完全一样**：各平台算法会检测搬运，标题、开头、配图要有差异
2. **不要追求完美**：先发 10 条烂的，比不发强 100 倍
3. **评论区话术不同**：
   - 小红书：引导私信/评论区留言
   - 公众号：引导关注 + 文末加微信
   - X：引导 follow + DM

### 产品层面

4. **合规风险**：账号转售可能违反平台 ToS，注意规避法律风险
5. **供应链风险**：上游渠道不稳会影响口碑，备好备用方案
6. **定价竞争**：赛道已有不少人，差异化很重要（服务 > 价格）
7. **售后压力**：账号类产品售后多，提前准备 FAQ 和自动化响应

### 心态层面

8. **冷启动没有反馈**：前 1-2 个月几乎没有点赞、没有粉丝、没有收入，这是正常的
9. **先活下来，再求规模**：不要一开始就三平台全铺，先单平台跑通再扩展
10. **固定时间创作**：每天花 30-60 分钟集中产出，不要用碎片时间硬挤

## 八、引流到私域的完整路径

```
公域引流 → 私域承接 → 信任建立 → 成交转化 → 复购裂变
```

| 环节 | 工具 | 作用 |
|-----|------|------|
| 个人号/企微 | 微信 | 1v1 信任建立 |
| 朋友圈 | 微信 | 日常种草、晒单、使用场景 |
| 微信群 | 社群 | 批量运营、答疑、促活 |
| 公众号 | 图文 | 教程沉淀、产品介绍、SEO |

### 成交转化

- **定价策略**：设置引流款（低价）+ 利润款（套餐/长期）
- **支付方式**：微信收款 → 后期可用小程序/有赞商城
- **交付方式**：自动化发卡（卡密系统）或手动交付

### 复购 + 裂变

- 老客续费优惠
- 推荐返现/分销机制
- 社群内定期推新品
