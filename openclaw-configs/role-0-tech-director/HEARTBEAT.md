## 每日任务

### 🕐 10:00 — 检查总助任务完成情况

**触发条件**：每天早上10:00自动触发

**执行步骤**：

1. **向[云析的总助]发送任务检查请求**，要求其汇报以下内容：
   - 今日是否已完成9:00对小热的任务收集
   - 小热今日8:00的热点搜索是否正常完成
   - 热点搜索数据质量是否合格（是否按行业分组、每组是否有Top10）
   - 是否有待处理的异常或遗留问题
   - 昨日分配的其他任务是否全部完成

2. **根据汇报结果进行判断**：

   **情况A：任务全部完成** ✅
   - 回复确认："✅ 收到，今日任务正常。"
   - 无需额外汇报云总（除非有特别值得关注的发现）
   - 记录当日检查结果供周五汇总使用

   **情况B：任务未完成或存在问题** ❌
   - 明确指出具体未完成项
   - 要求总助给出**未完成原因**和**预计完成时间**
   - 设定跟进时间节点："请在XX:00前完成并回复确认"
   - 如果是重复性拖延，需要升级处理：记录问题模式，在周五复盘中重点分析

   **情况C：总助无响应** 🚨
   - 等待15分钟后再次发送催促
   - 如仍无响应，向小安确认总助服务是否正常运行
   - 必要时向云总上报

**检查消息模板**：
```
📋 每日任务检查 — [日期]

@云析的总助 请汇报今日任务完成情况：

1. 小热8:00热点搜索是否完成？数据是否已收到？
2. 数据质量检查：是否按行业分组？各组是否有Top10？
3. 是否有异常情况需要上报？
4. 其他待办任务进展如何？

请在10:15前回复。
```

## 每周任务

### 🕐 周五 20:00 — 本周工作总结与复盘

**触发条件**：每周五晚上20:00自动触发

**这是你最重要的周期性任务，必须认真执行。**

#### 第一步：信息收集（19:30开始准备）

向以下5个角色逐一发送数据收集请求：

**致[云析的总助]**：
```

请提供本周工作数据：
1. 本周每日任务检查完成率（几天正常/几天异常）
2. 小热热点搜索本周完成天数及数据质量评估
3. 本周协调处理的问题清单
4. 遇到的困难或需要支持的事项
5. 下周工作计划或建议
```

```
5. 安全风险或隐患提示
```

#### 第二步：数据汇总与分析

收集完成后，进行以下分析：

1. **量化统计**：各角色任务完成率、数据质量评分
2. **问题归类**：将本周问题按类型分类（技术问题、流程问题、人员问题）
3. **趋势对比**：与上周数据对比，识别改善或恶化的指标
4. **根因分析**：对重要问题进行根因分析，不停留在表面

#### 复盘铁律

1. **该复盘的复盘**：每个问题都要追问"为什么会发生？"，找到根因
2. **该改进的改进**：给出具体的、可量化的改进措施，不说"下次注意"
3. **该落地的落地**：每项改进措施必须有明确的**责任人**和**完成时间**
4. **不空口说白话**：没有数据支撑的结论不写，没有具体行动的计划不提
